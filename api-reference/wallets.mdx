Virtual Wallets provide a flexible and secure way for organizations and their businesses to manage funds digitally. This document outlines key wallet operations including wallet creation, custom labeling, freezing/unfreezing, and wallet retrieval.

<ul>
  <li>
    <strong>Wallet Creation:</strong> Organizations and Organization businesses can create one or more virtual wallets based on specific operational needs. Each wallet serves as a dedicated account for managing transactions independently. This flexibility supports various use cases such as departmental budgets, customer accounts, or project-specific funds.
  </li>
  <li>
    <strong>Custom Wallet Labels:</strong> Every wallet can be assigned a unique custom label to easily identify its purpose or usage context. This helps in organizing and tracking multiple wallets under a single organization or business with ease.
  </li>
  <li>
    <strong>Freeze and Unfreeze Wallet:</strong> To enhance control and security, wallets can be temporarily frozenâ€”restricting all inflows and outflows. This is useful for fraud prevention, operational audits, or compliance checks. Once the need passes, wallets can be unfrozen to resume normal activity.
  </li>
  <li>
    <strong>Get All Wallets:</strong> Organizations and businesses can retrieve a paginated list of all virtual wallets associated with their profile. This allows for structured access to wallet data across various departments or service units.
  </li>
  <li>
    <strong>Get Wallet by ID:</strong> To access specific wallet details, organizations or businesses can query using the wallet ID. This operation returns all metadata and current status of the selected wallet.
  </li>
</ul>
